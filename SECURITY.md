# Security Policy

## Reporting a Vulnerability

We take the security of this documentation seriously. If you discover any sensitive information, proprietary data, or potentially harmful content (e.g., links to malware, unauthorized access methods) within this repository, please report it immediately.

### Preferred Method

Please **DO NOT** open a public GitHub issue. Instead, report the vulnerability directly to the repository maintainers via email:

*   **Email:** `security@bmcodex.com` (Placeholder - replace with actual security contact)
*   **Subject:** `SECURITY VULNERABILITY REPORT - [Repository Name]`

We will acknowledge your report within 48 hours and provide a detailed response within 7 days.

## Scope

This policy covers all content within the `bmw-oem-tools-documentation` repository.

## Sensitive Information Policy

This repository is intended to be a public, technical resource. As such, it **MUST NOT** contain:

*   **Proprietary BMW AG Data:** Any internal documents, source code, or unreleased specifications.
*   **Authentication Credentials:** Passwords, API keys, or private tokens.
*   **Personal Identifiable Information (PII):** Any data that could identify an individual.

If you find any such information, please follow the reporting procedure above. We will immediately remove the content and review the commit history.
